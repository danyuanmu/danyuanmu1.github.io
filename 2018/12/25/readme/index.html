用户关系测度项目

本项目基于用户在京东金融上的行为偏好，找到每个用户的关联度最高的1000用户，通过关联用户的偏好为该用户进行推荐

目录结构

目录 解释
log 日志
reuslt python执行结果
src 模型代码
src/get_features.sh 提取特征，结果存于hive表
src/data_haddle.sh 数据处理，结果存于hive表,为python的输入
src/cal_cosine.py python计算相关性，结果存于本地
src/result_load.sh 将python结果导入hive
src/get_Process.sh 流程汇总

使用指南

执行入口：

两个参数：月、日，作为字符串传入，该日期为选中当天的活跃用户作为训练样本；
期间所有的中间表及结果表都存于tech_dev中。

建模流程

src/get_features.sh

获取1天活跃用户的34个特征，input:month,day：
ep:

依次获取以下特征表：

  • 1、活跃用户列表 : userrelation_${month}${day}_active_pins
  • 2、大理财相关分类持仓表：userrelation_${month}${day}_fin_hold
  • 3、小金库分类持仓表：userrelation_${month}${day}_xjk_hold
  • 4、小白条当前欠款表：userrelation_${month}${day}_xbt_hold
  • 5、金条当前欠款表：userrelation_${month}${day}_jt_hold
  • 6、各类保险一年内购买金额表：userrelation_${month}${day}_bx_hold
  • 7、众筹、基金、大白条、银行+、小白定投相应金额表：userrelation_${month}${day}_join_hold

将上面的表合并为总的特征表：userrelation_${month}${day}_features，特征如下：

src/data_haddle.sh

这部分主要做两件事：1、列归一化;2、cosine相关性预处理，即提取出相关性公式的分母部分。

ep:

得到处理后的特征表：userrelation_${month}${day}_max_min_v11，即为python的input

src/cal_cosine.py

计算cosine相似性,并提取每个用户最相近的1000个用户，结果储存到本地。多进程执行。

ep:

src/result_load.sh

将python结果导入hive表，匹配上实际的pin,并排名。
ep:

userrelation_${month}${day}_rankresult：即最终的结果表；每个分区（pinid_part）储存10w用户的相关用户。
